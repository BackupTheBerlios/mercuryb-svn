Frequently Asked Questions

Here is a compiled list of question frequently asked in the support forum, these questions vary from help with installation to how to create a new skin.


Most common questions

I can't post on the board I just created
You need to create a category AND a forum to be able to post. You can do this through Admin CP -> Create a Forum. Be sure to use default, full or permissions from a group or forum already created. Otherwise you've created a forum fully locked down so even administrators cannot access it!

The board is blank or garbled
This could be because your server does not support GZip compression. You can turn this feature off by going into Admin CP -> Edit Board Settings and disable GZIP, you board should work as normal after this.

When people try and register they get a JpGraph error
This is because your web server does not have the required software to run anti-robot registration. Disable it through Admin CP -> Edit Board Settings, it is disabled by default.

How do I set the board to email registration validation?
Mercuryboard does not email newly registered users by default. You can turn it on in Admin CP -> Edit Board Settings -> Require users be validated by email. This will send an email to newly registered users with a link they must click before they become "active" members. Make sure your board email settings are correct!

The link it's emailing people is wrong. There's no domain name in it.
You need to enter the site address in Admin CP -> Edit Board Settings -> Location of Board. This must be set to the root of your domain address with no trailing slash. http://mysite.com is good. http://mysite.com/~geoff/forum is bad.

I can't login to my forum anymore!
Don't ban guests! It requires some very complicated SQL or editing PHP files to get back into your site! All groups except "Banned" should have "Use MercuryBoard" ticked or else you are banning that group from even login and logout!

How do I use the Admin CP?
MercuryBoard now has documentation on how to use the Admin CP, you can access this document by either going into the Admin CP and clicking on the help link at the top of the page, or click here.


Advanced questions

Can I put PHP code inside a HTML template?
You can get creative using the following syntax:(but without the spaces after '<')
< IF (php condition)>(html output)< /IF>

How do I disable the subscribe option?
You will need to edit the templates to remove the subscribe button. This isn't the best solution as someone who knows the url to type in could still subscribe. But it is the easiest.

Go into Admin CP -> Edit HTML Templates and select the skin to edit. Then go to Topics -> Topic View
Remove the last 3 lines
Save changes and continue to return to the templates list

Then go to Forums -> Forum View Look for

< p style='text-align:center'> < br />< a href='{$this->self}?a=cp&s=addsub&... snipped off rest of line for IE users < /p>

and remove that chunk. Save and you're done! This will not remove existing subscriptions. You'll need some database knowledge for that.

I installed a new skin but now can't edit/delete the images for it. Why can't I edit files I created?
It's all about permissions and ownership.

When you connect to the webserver through FTP you connect as a specific user (let's say halfling) and can create files that belong to halfling. You have permission, through FTP to change permissions, rename, delete, replace, etc.

Now when the web server creates a file (such as installing a skin from an mbs) it creates them as belonging to the user running webserver (nobody or a specific user like apache). That user can change permissions, rename, delete, replace, etc. (note: Yes there is a user called nobody)

The problem comes when these two don't overlap. Nobody creates a file that halfling can't delete. Or halfling creates a directory that nobody can't write to. Why we change permissions of other files and directories (typically to 777 or 775) is so nobody has access to write that file. Unfortunately the skin creation process doesn't (as yet) make the files so you can always FTP in to edit or delete them.

Solution? You can delete the skin through admincp. You can ask your web host administrator to change the ownership of the files. You can create a script to change the owner/permissions of the files. Or you can download the files, create a script to delete them, and then upload them as new.

Best and easiest is to talk to your hosting provider. They may fix things so it wont happen in the future.

I want to change how the board looks. Where do I do this?
Mercuryboard uses a skinning system that consists of templates, css and some graphics. You can find the css and graphics under the skins directory. The html templates can be edited through the admin cp. You should not need to edit PHP files to change the boards appearance. Also there are a lot of tricks you can do just in CSS.


Project questions

How can I join the Mercury Team?
The team itself is not currently looking for any new members, however, the project runs on what other people apart from the team contribute, so if you have a good feature then it might get added to a release.

Can I edit the source code?
Yes, 'Modding' the board is encouraged by the Mercury team so others can realize the power behind the project as well as post 'Mods' in the showcase forum of the main site. This allows others to use your Mods if you wish and is also a catalogue of what others have done, so take a look! Please take a look at MercuryBoard's coding standards and the GNU GPL License.
